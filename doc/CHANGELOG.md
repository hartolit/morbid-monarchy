- Fixed chunk-view seams at negative/global modulo boundaries by separating visible `window_origin` from the toroidal buffer head and rotating the head on window shifts.
- Aligned engine and shader indexing to sample the active grid via window-local coordinates plus `buffer_head`, preserving the toroidal grid without full rebuilds.
- Fixed `wgpu` validation panic (`opaque_mesh_pipeline` — binding missing from pipeline layout) by correcting the material bind group index in `world.wgsl` from `@group(2)` to `@group(3)`. Bevy 0.18 inserted `SetMeshViewBindingArrayBindGroup<1>` into the `DrawMaterial` command tuple, pushing the mesh bind group to slot 2 and the material bind group (`MATERIAL_BIND_GROUP_INDEX`) to slot 3. All three material bindings (`world_buffer`, `palette`, `window`) must use `@group(3)`.
- Fixed second `wgpu` validation panic (`Location[2] Float32x2 … is not provided by the previous stage`) by removing the unused `@location(2) uv: vec2<f32>` field from both `VertexInput` and `VertexOutput` in `world.wgsl`. The voxel mesh built by `build_voxel_grid` only supplies `POSITION`, `NORMAL`, `ATTRIBUTE_CELL_INDEX`, and `ATTRIBUTE_LAYER` — no UV attribute is ever inserted into the vertex buffer, so declaring `@location(2)` as a vertex shader input caused the pipeline validator to reject the vertex buffer layout mismatch. Also disabled `enable_prepass` and `enable_shadows` on `WorldMaterial` because the custom displacement vertex shader is incompatible with Bevy's standard prepass fragment interface.

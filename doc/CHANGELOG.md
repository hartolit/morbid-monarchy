- Fixed scrambled world on grid resize: `handle_simulation_resize` now iterates `new_active_view` during the RESTORE pass and sources chunks from `active_chunks`, `cached_chunks` (LRU pop), or cold `ChunkLoadRequest` — instead of blindly iterating `active_chunks` which missed every chunk evicted to the LRU cache during the same resize. Added `mut load_writer: MessageWriter<ChunkLoadRequest>` parameter to `handle_simulation_resize` for inline cold-miss requests. Changed `manager.active_view = None` to `manager.active_view = Some(new_active_view)` to prevent `manage_chunk_window` from double-requesting every chunk on the next tick.
- Fixed open-cube / pyramid artifact: `build_voxel_grid` was generating triangles in CW winding order from the outside of each face, causing Bevy/wgpu's CCW-front-face back-cull to discard all exterior geometry and render only inward-facing surfaces. Fixed by changing index order from `[0,1,2, 0,2,3]` to `[0,2,1, 0,3,2]` (CCW from outside).
- Fixed mesh not rebuilt after simulation resize: `sync_grid_rendering` only rebuilt the mesh when `count_vertices() == 0`, which is false for an existing mesh after a resize. Added `GridMeshSize` resource tracking last-built `(width, height)`; mesh is now rebuilt whenever `grid.width != mesh_size.width || grid.height != mesh_size.height`, preventing shader index mismatch after dimension changes.
- Added `doc/world_physics_vision.md` documenting the Algebraic Membrane Universe design: terrain height derived as `Z = H_MAX - atmos.state×scale - fluid.state×scale`, rationale for leaving `terrain.state` free for biology, water-flow-toward-higher-pressure CA rule, explosion/crater mechanics, and future considerations.
- Fixed chunk-view seams at negative/global modulo boundaries by separating visible `window_origin` from the toroidal buffer head and rotating the head on window shifts.
- Aligned engine and shader indexing to sample the active grid via window-local coordinates plus `buffer_head`, preserving the toroidal grid without full rebuilds.
- Fixed `wgpu` validation panic (`opaque_mesh_pipeline` — binding missing from pipeline layout) by correcting the material bind group index in `world.wgsl` from `@group(2)` to `@group(3)`. Bevy 0.18 inserted `SetMeshViewBindingArrayBindGroup<1>` into the `DrawMaterial` command tuple, pushing mesh to slot 2 and material (`MATERIAL_BIND_GROUP_INDEX = 3`) to slot 3.
- Fixed second `wgpu` validation panic (`Location[2] Float32x2 … is not provided by the previous stage`) by removing the dead `@location(2) uv` from both `VertexInput` (never supplied by the mesh) and `VertexOutput` (never written or read). Disabled `enable_prepass` and `enable_shadows` on `WorldMaterial` — the custom displacement vertex shader is incompatible with Bevy's standard prepass fragment interface.
- Fixed third `wgpu` validation panic (`Location[10] Uint32 … is not provided`) by adding `Material::specialize` to `WorldMaterial`, which explicitly registers `ATTRIBUTE_CELL_INDEX` and `ATTRIBUTE_LAYER` in the pipeline's vertex buffer layout. Without this, Bevy's default layout omits custom attributes and the pipeline validator rejects `@location(10)` / `@location(11)` as unsatisfied inputs.
- Fixed per-frame GPU lag: `sync_grid_rendering` was replacing the entire `ShaderStorageBuffer` allocation every frame the grid changed. Fixed by writing cell bytes directly into `buffer.data` in-place (`resize` + `copy_from_slice`), avoiding new GPU buffer object allocation.
- Fixed Y-axis mirror in rendered grid: `build_voxel_grid` was mapping engine row `y` to `offset_z = y`, causing the southern edge (row 0) to appear at the top of the screen. Fixed by flipping to `offset_z = (height - 1 - y)` so row 0 occupies the largest Z and appears at the screen bottom under a camera positioned at positive Z looking toward the origin.
- Replaced flat `FocalPoint` marker component with a full orbital camera state component (`anchor`, `yaw`, `pitch`, `distance`). Added `orbit_camera` (right-mouse-drag), `zoom_camera` (scroll wheel), and `apply_camera_transform` (single spherical-to-Cartesian derivation pass). WASD pan now translates the anchor instead of the raw `Transform`, keeping the orbital pose invariant during panning. Engine `WorldFocus` is now driven by the anchor position rather than the camera eye.

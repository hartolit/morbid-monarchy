- **Architectural Rewrite: 64-Bit `WorldCell` & Boundary-Aligned Bit Packing:** Radically optimized memory throughput by compressing `WorldCell` from 16 bytes down to exactly 8 bytes (`u64`). This doubles the CPU cache density to 8 cells per 64-byte cache line, significantly boosting parallel cellular automata performance in Rayon.
- **Strict 32-Bit Boundary Alignment for WGSL:** Structured the 64-bit `WorldCell` into two distinct 32-bit words to ensure zero-cost extraction in the GPU shader without cross-boundary bit-straddling. Word 0 (Geometry) packs Materials (5 bits each) and Elevation (17 bits). Word 1 (Physics/State) packs Fluid Volume (10 bits), Terrain State (8 bits), Visual Variants (6 bits), Momentum Flags (4 bits), and Awake Flags (4 bits).
- **Physics Inversion & Deletion of "Atmospheric Pressure":** Removed the algebraic "pressure-based" terrain height system. The grid now directly stores a 17-bit `elevation` value (max height 131,071), giving immense verticality. Fluid flow logic (`liquid_sim.rs`) was inverted to flow naturally toward lower absolute elevation rather than higher atmospheric pressure.
- **Decoupled Material Domains:** Split the global 8-bit `MaterialId` into tightly packed 5-bit `TerrainMat`, `FluidMat`, and `SurfaceMat` enums (max 31 variants each) to fit the new geometry word. The `world.wgsl` shader offsets these local IDs into the global 256-color palette to resolve rendering colors seamlessly.
- **Solved "Water Stacking" Limit:** By allocating 10 bits to `fluid_vol`, liquid depth can now reach up to 1,023 units per cell (previously clamped at 255), fixing volume-loss bugs during massive parallel fluid collisions.
- **Updated Dev Tools:** Migrated the `GridBrush` tools to mutate absolute elevation (`RaiseTerrain`, `LowerTerrain`) instead of abstract gas pressures, aligning the developer UI with the new physics logic.
